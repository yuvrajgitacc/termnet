1. Getting Started

Access the platform via your deployment URL or run locally with python app.py

signup <your_username> Example: signup yuvraj — Creates your profile on the system.
login <your_username> Example: login yuvraj — Authenticates your session. Required every time you connect.

2. Rooms & Communication

Channel Management

list Displays all active communication rooms.
create <room_name> - <password> Example: create studygroup - 1234 — Deploy a password-protected room.
jump <room_name> - <password> Connects you to a room. Others in the room will be notified.
destroy <room_name> Deletes a room and all associated files. Cannot destroy 'lobby'.
who NEW Shows all users currently connected to your room.
exit Disconnects from the current room and returns to home.

Messaging

msg <text> Broadcasts a message to everyone in your current room.
clear Wipes the terminal buffer for a clean workspace.

3. File Sharing Protocol

share IMPROVED Upload a file to the Public Room. Uses chunked transfer with a live progress bar. You can also drag-and-drop files onto the terminal.
vault Upload a file to Private Storage. Only you can access this.
fetch <filename> IMPROVED Example: fetch notes.pdf — Downloads with chunked transfer and a progress bar. Searches both room and your private vault.
profile Displays your identity card and a list of all your uploaded files with sizes.

5. Important Considerations

Critical: Free Hosting Limitations

  • Cold Start: After 15 mins of inactivity, the server sleeps. The first request takes 30-60 seconds to load.
  • Ephemeral Storage: All data (accounts, messages, files) is wiped clean whenever the server restarts or sleeps.
  • Auto-Cleanup: Files older than 24 hours are automatically deleted.
  • File Size: Maximum upload size is 50MB per file.
